# Запись на звонок

## Суть

Упрощённый сервис бронирования времени для встреч, вдохновлённый Cal.com.

## Проблема

Владельцу нужно опубликовать своё доступное время, а гостю — быстро выбрать слот и записаться. Без регистрации, без лишних шагов.

## Роли

В проекте две роли: **владелец календаря** и **гость**.

- Регистрации и авторизации нет. Владелец календаря — один заранее заданный профиль.
- Гость бронирует слоты без создания аккаунта и без входа в систему.

## Типы событий

Владелец календаря создаёт типы событий. Для каждого типа задаётся:

- Название
- Описание
- Длительность в минутах

Примеры: «Консультация» (60 мин), «Быстрый звонок» (15 мин), «Демо продукта» (30 мин).

## Страницы

### Лендинг

Приветственная страница с кратким описанием сервиса и кнопкой «Записаться», которая ведёт на страницу выбора типа события.

### Виды брони

Страница со списком доступных типов событий. Для каждого отображается:

- Название
- Описание
- Длительность

Гость выбирает тип события и переходит к календарю.

### Запись на звонок

Основная страница с тремя блоками:

- **Информация** — выбранный тип события, дата, время, количество свободных слотов, длительность
- **Календарь** — месячный вид с навигацией по месяцам; у дат с доступными слотами отображается количество свободных; прошедшие даты недоступны для выбора
- **Статус слотов** — список слотов на выбранный день с отметкой «Свободно» / «Занято»; если все слоты заняты — отображается сообщение «Нет слотов на этот день»

Доступные слоты рассчитываются с учётом длительности выбранного типа события.

Гость выбирает дату, затем слот, нажимает «Продолжить».

### Подтверждение записи

Форма с полями:

- Имя
- Email

Кнопка «Подтвердить запись». Есть возможность вернуться и изменить выбранный слот.

Валидация:

- Имя — обязательное поле
- Email — обязательное поле, проверка формата

Если слот был занят другим гостем (конфликт) — показать ошибку и предложить выбрать другой слот.

### Экран успеха

Сообщение «Бронь подтверждена. До встречи!» и кнопка «Забронировать ещё».

### Предстоящие события

Список всех записей по всем типам событий в одном списке. Каждая запись содержит:

- Тип события (название)
- Имя гостя
- Email
- Дата и время слота
- Дата создания записи

## Навигация

В шапке два пункта:

- «Записаться» — ведёт на страницу выбора типа события
- «Предстоящие события» — ведёт на список записей

## Основной сценарий (гость)

1. Открывает лендинг, нажимает «Записаться»
2. Видит список типов событий, выбирает нужный
3. Выбирает дату в календаре
4. Выбирает свободный слот из списка
5. Нажимает «Продолжить»
6. Вводит имя и email
7. Нажимает «Подтвердить запись»
8. Видит подтверждение

## Правило занятости

На одно и то же время нельзя создать две записи, даже если это разные типы событий. При пересечении временных интервалов (с учётом длительности) бронирование отклоняется.

## Параметры расписания

- Рабочие часы: 09:00–18:00
- Длительность слотов зависит от выбранного типа события
- Количество доступных слотов в день рассчитывается динамически на основе длительности типа события и уже существующих бронирований

## Ограничения MVP

- Нет авторизации и личных кабинетов
- Нет интеграций с внешними календарями
- Один владелец, одно расписание
- Расписание одинаковое на каждый день (09:00–18:00)
- Страница «Предстоящие события» доступна всем без авторизации (учебный проект)
